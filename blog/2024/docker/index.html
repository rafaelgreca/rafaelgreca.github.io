Docker study notes

February 12, 2024

2024   ·   docker   model-deployment   study-notes     ·   docker   model-deployment   study-notes  

Introduction

“Docker is a virtualization software that makes developing and deploying applications much easier. It packages application with all the necessary dependencies, configuration, system tools and runtime.”

Vocabulary

Docker Images

It’s an executable application artifact, which includes the application source code, the complete environment configuration, environment variables, directories, files, and so on.

To build your own image, you create a Dockerfile with a simple syntax for defining the steps needed to create the image and run it. Each instruction in a Dockerfile creates a layer in the image. When you change the Dockerfile and rebuild the image, only those layers which have changed are rebuilt.

Docker Containers

It’s the running instance of an image (that’s when the container environment is created).

You can connect a container to one or more networks, attach storage to it, or even create a new image based on its current state. A container is defined by its image as well as any configuration options you provide to it when you create or start it. When a container is removed, any changes to its state that aren’t stored in persistent storage disappear.

Docker Registry

It’s a storage and distribution system for Docker images.

When you use the docker pull or docker run commands, Docker pulls the required images from your configured registry. When you use the docker push command, Docker pushes your image to your configured registry.

Docker Repository

It’s a collection of related images with the same name but with different versions.

Docker Network

Container networking refers to the ability for containers to connect to and communicate with each other, or to non-Docker workloads.

Containers have networking enabled by default, and they can make outgoing connections. A container has no information about what kind of network it’s attached to, or whether their peers are also Docker workloads or not. A container only sees a network interface with an IP address, a gateway, a routing table, DNS services, and other networking details. That is, unless the container uses the none network driver.

Docker Volume

Volumes are the preferred mechanism for persisting data generated by and used by Docker containers. While bind mounts are dependent on the directory structure and OS of the host machine, volumes are completely managed by Docker.

Volumes are often a better choice than persisting data in a container’s writable layer, because a volume doesn’t increase the size of the containers using it, and the volume’s contents exist outside the lifecycle of a given container.

Docker Compose

Docker Compose is a tool for defining and running multi-container applications. It is the key to unlocking a streamlined and efficient development and deployment experience.

Compose simplifies the control of your entire application stack, making it easy to manage services, networks, and volumes in a single, comprehensible YAML configuration file. Then, with a single command, you create and start all the services from your configuration file.

Structure of Docker compose yaml file :

  • version (version of docker-compose)
  • services (all the services/containers that you want to run as a part of this compose file)
    • the container name
      • image: name of the image
      • ports: the binding ports (which follows the same structure as in the docker run command)
      • environment: the environment variables
      • depends_on: a list of the services that this containers depends on

Docker Swarm

It is used to combine multiple Docker machines together in a single cluster .

Dockerfile

General structure of Dockerfile :

  • Dockerfiles must begin with a FROM instruction (build this image from the specified image)
  • The COPY command will copy files or directories from and adds them to the filesystem of the container at the path (it's executed on the host)
  • The WORKDIR command will set the working directory for all following commands (works like the cd command)
  • The RUN command will execute any command in a shell inside the container environment
  • The CMD command will execute the instruction when a Docker container starts (there can only be one CMD instruction in a Dockerfile and it must be the last line in the Dockerfile)

Instructions :

  • ADD: Add local or remote files and directories
  • ARG: Use build-time variables
  • CMD: Specify default commands
  • COPY: Copy files and directories
  • ENTRYPOINT: Specify default executable
  • ENV: Set environment variables
  • EXPOSE: Describe which ports your application is listening on
  • FROM: Create a new build stage from a base image
  • HEALTHCHECK: Check a container’s health on startup
  • LABEL: Add metadata to an image
  • MAINTAINER: Specify the author of an image
  • ONBUILD: Specify instructions for when the image is used in a build
  • RUN: Execute build commands
  • SHELL: Set the default shell of an image
  • STOPSIGNAL: Specify the system call signal for exiting a container
  • USER: Set user and group ID
  • VOLUME: Create volume mounts
  • WORKDIR: Change working directory

Notes

1) List all Docker images

docker images

2) List running Docker containers

docker ps

Using the argument -a will list all the containers, including those that are not running. Example:

docker ps -a

3) Pull an image from a registry

docker pull <name>:<tag>

4) Creates a container from a given image and starts it

docker run <name>:<tag>

Using the argument -d or --detach the container will run in background (the container ID will be printed). Example:

docker run -d <name>:<tag>

Using the argument -p or --publish will publish a container’s port to the host. Example:

docker run -p <host_port>:<container_port> <name>:<tag>

Using the argument --name will assign a name to the container. Example:

docker run --name <container_name> <name>:<tag>

Using the argument -v or --volume will attach a volume to the Docker container. Example:

docker run -v <host_dir>:<container_dir> <name>:<tag>

Using the argument --network will attach a network to the Docker container. Example:

docker run --network <network> <name>:<tag>

Using the argument --restart will change the restart policy. Example:

docker run --restart <option> <name>:<tag>

Using the argument --link will link a Docker container with another (adds an entrypoint). Example:

docker run --link <container>:<container> <name>:<tag>

Using the argument --cpus will limit the CPU usage. Example:

docker run --cpus=<percentage_value> <name>:<tag>

Using the argument --memory will limit the memory usage. Example:

docker run --memory=<value> <name>:<tag>

Using the argument -it will run the Docker container in the interactive mode. Example:

docker run -it <network> <name>:<tag>

Example of how to append commands:

docker run <name>:<tag> <command>

Using the argument -e will pass an argument to Docker container. Example:

docker run -e <variable>=<value> <name>:<tag>

Example of how to append commands:

docker run <name>:<tag> <command>

5) View the logs from the service running inside the container

docker logs <container>

6) Stop a running container

docker stop <container>

7) Start a stopped container

docker start <container>

8) Build a Docker image

docker build

Using the argument -t or --tag will set a name and optionally a tag in the “name:tag” format. Example:

docker build -t  <name>:<tag> .

9) Search Docker images

docker search <name>

10) Run a container with Docker compose

docker compose up

11) Shut down a running container with Docker compose

docker compose down

12) Remove all stopped Docker containers

docker container prune

13) Change the Docker image’s name and tag

docker image tag <name>:<tag> <new_name>:<new_tag>

14) Push the Docker image to a Docker registry (Docker Hub)

docker push <name>:<tag>

15) Remove a Docker image

docker rmi <name>

16) Remove a Docker container

docker rm <name>

17) Get to the terminal of a running Docker container

docker exec -it <name> /bin/bash

or

docker exec -it <name> sh

18) Create a Docker network

docker network create <name>

Using the argument -d or --drive will set the network’s driver (bridge, host or none). Example:

docker network create -d  <driver>

Using the argument --subnet will set the network’s subnet. Example:

docker network create --subnet <subnet>

Using the argument --gateway will set the network’s gateway. Example:

docker network create --gateway <gateway>

19) List all Docker networks

docker network ls

20) Create a Docker volume

docker volume create <name>

21) Start a container with Docker compose

docker compose start

22) Stop a running container with Docker compose

docker compose stop

23) Attach a running container (this lets you view its output or control it interactively, as though the commands were running directly in your terminal.)

docker attach <container>

24) Inspect a Docker image/container/network

docker inspect <image/container/network>

25) Show the Docker image’s history

docker history <image>

26) Change the image tag

docker image tag <image_name> <new_image_name>

27) Log in into your account (mandatory step to push a image to Docker Hub)

docker login

28) Checking memory usage

docker system df -v

29) Docker orchestration with Docker Swarm (similar parameters as Docker run)

docker service create --replicas=<value> <image>

30) Initialize Docker Swarm manager

docker swarm init

31) Run Docker Swarm workers (to join the manager)

docker swarm join --token <token>

32) Full Dockerfile example:

# building the Docker image on top of the Python 3.11
# slim (a ligther version) image
FROM python:3.11-slim

# creating a new folder inside the container
RUN mkdir /disaster-tweet-classification

# changing the working directory to the folder
# that we have just created
WORKDIR /disaster-tweet-classification

# installing pip
RUN pip install --no-cache-dir -U pip

# copying all the files to the working directory
COPY . .

# installing all the requirements using pip
RUN pip install -r requirements.txt

# running the api script
CMD ["python", "api.py"]

33) Full Docker compose example:

version: '3.3'

services:
    api:
     # using the Dockerfile to build the image
     build: .
    ports:
     # binding the ports
     - "8000:8000"
